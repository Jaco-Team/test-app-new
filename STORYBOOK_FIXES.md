# Storybook-First План (Источник Дизайна)

Дата обновления: 2026-05-06

## Цель

Сделать Storybook единственным источником правды по дизайну:

1. Сначала изменение дизайна/состояний делается в Storybook.
2. Потом это изменение переносится в основной проект через те же компоненты.
3. Запрещаем дальнейшее развитие «копий» компонентов в `stories/*`.

## Что это меняет в процессе

- Было: дизайн жил в `modules/*`, Storybook частично отставал.
- Должно быть: дизайн живет в Storybook (на базе реальных компонентов), проект только потребляет и собирает экраны.

## Текущее состояние (кратко)

- В проекте `63` story-файла, `210` story-variant, `63` docs-страницы.
- Большая часть историй сделана на локальных копиях из `stories/*`, а не на реальных `modules/*`/`ui/*`.
- Есть визуальный дрейф между Storybook и реальной страницей `/togliatti`.

## Критерии готовности этапа «Storybook стал источником дизайна»

- Не менее 90% ключевых UI-экранов в Storybook рендерятся через реальные компоненты.
- Новые дизайн-изменения не вносятся напрямую в legacy story-копии.
- Для всех критичных экранов есть baseline в visual regression.
- Команда в PR следует процессу: Storybook -> проект, а не наоборот.
